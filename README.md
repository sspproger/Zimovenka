# Зимовенька — Усадьба Лыковых

Статический одностраничный сайт семейного производства на Astro 7 и TypeScript. Основное действие — переход в Telegram или во ВКонтакте для просмотра актуального ассортимента и заказа. Корзины, оплаты, форм и вымышленных товарных данных нет.

## Запуск

Требуется Node.js 22.12+.

```bash
npm install
npm run dev
npm run build
npm run preview
```

Сборка находится в `dist/` и не требует постоянно работающего Node.js.

## Структура

- `src/assets/` — исходная фирменная графика;
- `src/components/` — секции лендинга и SEO-компоненты;
- `src/data/` — централизованные тексты, ссылки, товары и FAQ;
- `src/content/{products,articles,stories}/` — Content Collections;
- `src/layouts/` — общий HTML-каркас;
- `src/pages/` — главная, 404, политика, robots и RSS;
- `src/styles/` — глобальные mobile-first стили;
- `public/` — favicon, Open Graph и web manifest.

## Редактирование

Основные ссылки, контакты, метаданные и плейсхолдеры находятся в `src/data/site.ts`. Категории и список неподтверждённых товаров — в `src/data/products.ts`. FAQ редактируется в `src/data/faq.ts`; эти же ответы автоматически попадают в FAQPage JSON-LD.

Для замены изображений сохраните пропорции и имена файлов в `src/assets/`. Astro создаёт современные форматы для изображений, импортированных компонентами. После замены обновите `public/og-image.png` и `public/apple-touch-icon.png`.

## Добавление материалов

Создайте Markdown-файл в нужной коллекции. Пример статьи:

```md
---
title: "Заголовок"
description: "Краткое описание"
publishDate: 2026-08-01
draft: true
---
Текст материала.
```

Черновики не выводятся в RSS. Перед публикацией создайте маршрут статьи, проверьте факты и установите `draft: false`. Для продукта дополнительно доступны `available` и `price`; Product/Offer разметку добавляйте только при подтверждённых данных.

## SEO и аналитика

Задайте канонический домен перед сборкой: PowerShell — `$env:SITE_URL='https://example.ru'; npm run build`. Sitemap и robots создаются автоматически, RSS доступен по `/rss.xml`.

Идентификаторы аналитики не заданы, поэтому счётчики не подключены. События обозначены атрибутами `data-event`. После получения ID добавьте один отложенный скрипт в `BaseLayout.astro`, отправляйте события только при наличии счётчика и дополните политику конфиденциальности. Коды подтверждения Яндекс Вебмастера и Google Search Console добавляются как meta-теги в `BaseLayout.astro`.

## Развёртывание

- Netlify / Vercel / Cloudflare Pages: build command `npm run build`, output `dist`;
- обычный хостинг: загрузите содержимое `dist/` в корень сайта;
- Nginx: `root /path/to/dist; try_files $uri $uri/ =404;`.

После публикации проверьте canonical, sitemap, robots, Open Graph, JSON-LD, редиректы HTTPS и Lighthouse. Добавьте сайт в Яндекс Вебмастер и Google Search Console. Резервное копирование — репозиторий плюс отдельная копия исходных бренд-файлов.

## Данные, которые необходимо получить у семьи Лыковых

- достоверная история, год основания, состав семьи и распределение обязанностей;
- описание производства, рецептур и подтверждённый перечень продукции;
- составы, сроки и условия хранения, цены и сезонность;
- география доставки, адреса точек, карты и график;
- телефон, email, адрес, режим работы и реквизиты;
- юридическая информация и данные оператора персональных данных;
- реальные фотографии, разрешение на их публикацию, подтверждённые цитаты и отзывы.

Все поля в квадратных скобках — плейсхолдеры. Все категории и конкретные продукты из `unconfirmedProducts` пока неподтверждённые и не заявляются как доступные.

Одностраничный сайт — первый этап. Продвижение по отдельным товарным запросам потребует подтверждённых продуктовых страниц, полезных статей и устойчивой публикации оригинального контента. Позиции в поиске не гарантируются.
